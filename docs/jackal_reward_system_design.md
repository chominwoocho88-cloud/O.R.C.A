# JACKAL 보상 기반 학습 시스템 설계안

> 작성: 2026-06-12 · 상태: 설계 검토 단계 (코드 미반영)
> 대상: `apps/jackal/tracker.py`, `apps/jackal/evolution.py`, `jackal/thresholds.py`, `jackal/jackal_weights.json`

## 1. 배경 — 현행 학습의 한계

현재 tracker의 가중치 학습(`tracker.py: _update_weights`, L233~)은 다음 구조다.

```
적중 판정: peak_pct >= +1.0% (이진)
조정 규칙: 누적 정확도 >= 70% → w += 0.04
          누적 정확도 <= 40% → w -= 0.03
```

문제점 세 가지.

1. **보상이 이진**: +12% 적중과 +1.1% 턱걸이 적중이 동일한 상을 받는다. -0.2% 미적중과 -9% 손실도 동일한 벌. 신호의 "질"이 학습에 반영되지 않는다.
2. **누적 정확도의 영구 기억**: `signal_accuracy`가 전 기간 누적이라 과거 성과가 영원히 남는다. 시장 레짐이 바뀌어 신호가 죽어도 누적 정확도가 70% 위에 있는 동안 계속 상을 받는다 (시뮬레이션 §7에서 확인: 붕괴 신호가 w=2.5 상한에 고착).
3. **고정폭 조정**: 결과의 크기·확신도와 무관한 ±0.04/0.03 스텝. 한 번 바닥(0.3)에 떨어진 신호는 사실상 부활 경로가 없다.

## 2. 설계 목표

- 보상의 **크기**가 학습에 반영될 것 (큰 적중 = 큰 상)
- 보상이 **리스크 조정**될 것 — 변동성 큰 잡주가 우연히 낸 +5%보다 저변동 종목의 안정적 +2.5%가 더 큰 상을 받아야 함. 순수 수익률 보상은 시스템을 복권형 종목 선호로 진화시킨다
- **손실 회피 비대칭**: 같은 크기의 벌이 상보다 아프게 (드로다운 방어 학습)
- **최근성**: 옛 성과는 잊혀지고, 벌 받은 신호도 회복 가능할 것 (용서)
- 기존 구조(가중치 스칼라, 상하한 0.3~2.5, Evolution 연계)는 유지 — 전면 RL이 아닌 하이브리드

## 3. 보상 함수

확정된 entry 1건마다 보상 `r ∈ [-1.5, +1.0]`을 계산한다.

### 3.1 유효 수익률 (effective return)

```
적중 시   : eff = peak_pct                  (스윙 최고 수익률)
미적중 시 : eff = outcome_pct               (5일 수익률, 대개 음수)
```

미적중에 `outcome_pct`를 쓰는 이유: 이진 0이 아니라 "얼마나 틀렸는지"가 벌의 크기가 된다.

### 3.2 리스크 조정

```
raj = eff / (vol_d × √peak_day)
```

- `vol_d`: 추적 기간(7거래일) 종가의 일간 수익률 표준편차(%). tracker가 이미 받아오는 종가 시리즈로 계산 — **추가 API 비용 0**
- **floor 필수**: `vol_d = max(vol_d, 0.5)`. 분모가 0에 가까워지면 보상이 폭주(reward hacking)
- `√peak_day` 시간 정규화: 7일 걸려 도달한 +3%보다 1일 만의 +3%가 더 큰 상

### 3.3 경계화 + 손실 비대칭

```
r = tanh(raj / k)                  # k = 1.5, r ∈ (-1, 1)
if r < 0: r *= λ                   # λ = 1.5 (손실 회피 계수)
r = max(r, -1.5)
```

`tanh`로 단일 이벤트가 가중치를 폭주시키는 것을 방지. 음수 보상은 1.5배 증폭.

## 4. 가중치 갱신 규칙

신호별로 `_update_weights` 내 고정 스텝을 다음으로 교체.

```
warm = min(1, n / 5)                          # 워밍업 (기존 MIN_SAMPLES_ADJ 계승)
w ← clip( w + η·r·warm + κ·(1.0 − w),  0.3, 2.5 )
```

- `η = 0.06` (학습률): 보상 비례 스텝. 만점 보상이어도 1회 +0.06 — 기존 ±0.04와 같은 자릿수라 안정성 유지
- `κ = 0.02` (평균 회귀): 매 갱신 시 1.0을 향해 2% 끌림. 효과 두 가지 — (a) 상한·하한 고착 방지, (b) 바닥에 떨어진 신호의 자연 부활 경로 = "용서". 성과가 계속 좋아야만 높은 가중치가 유지된다 (유기적)

### 4.1 EMA 보상 추적 (통계용)

`signal_accuracy`는 호환성 위해 유지하되, 신호별 EMA 보상을 추가한다.

```
ema_r ← (1−α)·ema_r + α·r          # α = 0.15 (≈ 최근 13건 반감)
```

`ema_r`은 가중치 갱신엔 직접 쓰지 않고 (이중 적용 방지), Evolution의 Claude 컨텍스트와 룰 자동 비활성화 판단에 쓴다.

## 7. 시뮬레이션 검증

4개 가상 신호 × 200건, 동일 난수로 기존/신규 방식 비교 (`scripts` 미포함, 검증 스크립트는 설계 첨부본 참고).

| 신호 프로필 | 기존 최종 w | 신규 최종 w | EMA r |
|---|---|---|---|
| strong_calm (μ+2.5%, 저변동, 적중 70%) | 1.48 | **2.14** | +0.44 |
| lottery (μ+2.5%, 고변동, 적중 45%) | 0.30 | **1.16** | +0.21 |
| weak (μ+0.5%, 적중 35%) | 0.30 | **1.02** | -0.07 |
| regime_flip (t=100에 적중 65%→25% 붕괴) | **2.50 고착** | **1.20로 감쇠** | +0.03 |

해석: 기존 방식은 붕괴 신호를 상한에 영구 고착시키고(누적 정확도 기억), 부진 신호를 바닥에 영구 추방한다. 신규 방식은 우량 신호를 크게 키우고, 복권형은 우량의 54% 수준으로 차등하며, 레짐 붕괴에 감쇠로 반응하고, 부진 신호도 1.0 근방에서 재기회를 유지한다.

## 8. 가드레일 및 롤아웃

1. **Phase 0 — Shadow (2주)**: `enabled: False` 상태로 reward·ema_r·가상 가중치를 계산해 `jackal_weights.json`에 `shadow_weights`로만 기록. 실제 스코어링은 기존 가중치 사용. 기존 shadow 패턴(`memory_context_shadow.log`, `contract_shadow_audit.log`)과 동일한 운영 방식
2. **Phase 1 — 전환**: shadow 가중치와 실가중치의 괴리·안정성 확인 후 `enabled: True`. 텔레그램 요약에 reward 변경 로그 포함
3. **Phase 2 — Evolution 연계**: §6의 evolution.py 변경 적용
4. **Phase 3 (선택) — 서프라이즈·탐험**: hunt 시점 확신도(스코어)를 저장해 `r' = r − 기대치`로 calibration 학습. 바닥 가중치 신호를 주기적으로 shadow 부활시켜 재평가

주의사항: 하루 ~5종목이면 신호당 주 1~3건의 보상 샘플. η를 키우고 싶어도 **표본 노이즈가 우선 제약**이므로 파라미터 튜닝은 shadow 기간 실데이터로만 판단할 것. `vol_floor` 제거 금지 (보상 폭주). κ 제거 금지 (고착 재발).

## 9. 요약

상/벌 개념 자체보다 중요한 것은 (1) 보상의 크기·리스크 반영, (2) 망각, (3) 용서다. 이 세 가지가 들어가면 가중치가 시장과 함께 숨 쉬는 구조가 되고, Evolution의 Claude 학습 루프에도 이진 적중률보다 훨씬 풍부한 신호가 공급된다.
